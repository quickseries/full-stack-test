Question 2:

Create a function that can find the Nth smallest number in an array. The array can be very very big, so you are not allowed using the Array.sort() method. Try to build a function that would perform fast. Also, explain why using Array.sort() isn't a good idea when sorting large arrays.


Why we can use Array.sort() for long array?
Answer: Array.sort() function compares each value with next value and do the sorting. Which is very costly in terms of performance. A big array consumes more memory and function require more computational power to compare all values to sorting. I compared few algorithms selection sort, merge sort, and heap sort. Heap sort has lowest complexity and it's best for the given problem. Selection sort also performing well when we have nth number very small( may be between 1 to 15).

I tested all three algorithm with same test data, If my nth value is near to 10 then Selection sort is the best solution. For any other nth value I would prefer to go with heap sort.
